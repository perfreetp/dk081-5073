# 错误码说明文档

## 概述

本文档列出了系统所有的错误码及其含义、建议处理方式。所有错误响应统一返回 `Result<T>` 格式，包含 `code`、`message`、`method`、`timestamp` 字段。

---

## 成功码

| 错误码 | 说明 | 建议处理方式 |
|--------|------|--------------|
| 200 | 操作成功 | 正常处理响应数据 |

---

## 认证授权类 (401xx)

| 错误码 | 枚举值 | 说明 | 建议处理方式 |
|--------|--------|------|--------------|
| 40101 | TOKEN_EXPIRED | 登录已过期，请重新登录 | 清除本地Token，跳转至登录页 |
| 40102 | TOKEN_INVALID | Token无效 | 检查Token格式和签名，重新登录获取 |
| 40103 | TOKEN_EMPTY | Token为空 | 请求头中添加Authorization字段 |
| 40104 | ACCOUNT_LOCKED | 账号已锁定 | 联系管理员解锁账号 |
| 40105 | ACCOUNT_EXPIRED | 账号已过期 | 联系管理员续期账号 |
| 40106 | PASSWORD_ERROR | 密码错误 | 检查密码是否正确，或找回密码 |
| 40107 | PASSWORD_EXPIRED | 密码已过期 | 修改密码后重新登录 |

---

|--------|--------|------|--------------|

| 40303 | PERMISSION_API_DENIED | 无接口权限 | 联系管理员分配接口权限 |

## 参数校验类 (400xx)

| 错误码 | 枚举值 | 说明 | 建议处理方式 |
|--------|--------|------|--------------|
| 40001 | PARAM_MISSING | 参数缺失 | 检查必填参数是否已填写 |
| 40002 | PARAM_FORMAT_ERROR | 参数格式错误 | 检查参数格式（如日期、邮箱、手机号等） |
| 40003 | PARAM_RANGE_ERROR | 参数范围错误 | 检查参数取值范围 |
| 40004 | PARAM_VALIDATE_ERROR | 参数校验失败 | 查看具体错误信息，修正参数 |
| 40005 | REQUEST_BODY_EMPTY | 请求体为空 | 确保POST/PUT请求包含请求体 |

|--------|--------|------|--------------|
| 30001 | ALARM_ALREADY_HANDLED | 警情已处置 | 该警情已完成处置，无需重复操作 |
| 30002 | ALARM_ALREADY_CLOSED | 警情已关闭 | 该警情已关闭，无法进行操作 |
| 30003 | ALARM_STATUS_NOT_ALLOWED | 警情状态不允许 | 检查当前警情状态是否支持该操作 |

| 30005 | ALARM_CANNOT_MERGE_SELF | 不能合并自己 | 主警情不能与自己合并 |

## 响应格式示例

### 成功响应
```json
{
  "code": 200,
  "message": "操作成功",
  "data": { ... },
  "timestamp": 1718745600000,
  "method": "POST /api/auth/login"
}
```

### 错误响应
```json
{
  "code": 40101,
  "message": "登录已过期，请重新登录",
  "data": null,
  "timestamp": 1718745600000,
  "method": "GET /api/alarm/record/page"
}
```

---

## 使用建议

1. **前端处理**：根据错误码进行统一的错误提示和页面跳转
2. **日志排查**：`method` 字段可帮助快速定位出错的接口
3. **动态参数**：部分错误消息支持动态参数，如 `参数缺失: username`
4. **国际化**：错误码可作为国际化消息的key，便于多语言支持
